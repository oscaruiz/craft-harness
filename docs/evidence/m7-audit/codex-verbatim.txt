1. Medium — Acceptance-report duplicates are accepted — PARTIAL

The parser validates JSON structure, non-empty scenario names, and known statuses, but does not reject duplicate scenario IDs or enforce the documented ID grammar ([bin/parse-accept-report.bb](D:/Dev/craft-harness/bin/parse-accept-report.bb:24), [bin/parse-accept-report.bb](D:/Dev/craft-harness/bin/parse-accept-report.bb:29)). `run-six` then records a scenario as passed if any record for that ID says `passed` ([bin/run-six](D:/Dev/craft-harness/bin/run-six:384)). Consequently:

```json
{"scenario":"SUT-1","status":"failed"}
{"scenario":"SUT-1","status":"passed"}
```

passes the gate, as does the reverse order. `inspect-run` repeats the same existential logic ([bin/inspect-run](D:/Dev/craft-harness/bin/inspect-run:129)). This does not make the fixture green by fiat, but a real acceptance adapter emitting retries or duplicate results could conceal a failure.
